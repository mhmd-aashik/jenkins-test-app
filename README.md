# NestJS + Drizzle ORM + Docker + Jenkins CI/CD Pipeline

A professional, production-ready Task CRUD application built with NestJS, Drizzle ORM, and PostgreSQL. It features full Docker containerization, a declarative Jenkins CI/CD pipeline, automated testing (unit and E2E), and Slack status notifications.

---

## 1. Project Architecture

The codebase follows a modular NestJS structure:

```text
jenkins-test-app/
├── drizzle/                     # SQL migration scripts generated by drizzle-kit
├── src/
│   ├── config/                  # Environment variables schema and validation (class-validator)
│   │   └── env.config.ts
│   ├── database/                # Database modules, Drizzle providers, and schemas
│   │   ├── database.module.ts
│   │   ├── database.provider.ts
│   │   └── schema.ts
│   ├── tasks/                   # Tasks CRUD resource module
│   │   ├── dto/                 # Input validation DTOs (CreateTaskDto, UpdateTaskDto)
│   │   ├── tasks.controller.ts  # HTTP routes handler
│   │   ├── tasks.service.ts     # Business logic and Drizzle ORM queries
│   │   └── tasks.module.ts
│   ├── app.module.ts            # Root application module wiring
│   └── main.ts                  # Entrypoint bootstrapping global pipes & shutdown hooks
├── test/                        # E2E test suites running against test containers
│   ├── app.e2e-spec.ts
│   ├── tasks.e2e-spec.ts
│   └── jest-e2e.json
├── Dockerfile                   # Multi-stage production container definition
├── Dockerfile.dev               # Dev container mounting host folder (hot-reload)
├── docker-compose.yml           # Dev stack (App + PostgreSQL)
├── docker-compose.test.yml      # CI/CD test PostgreSQL database
├── drizzle.config.ts            # Drizzle CLI migration manager config
└── Jenkinsfile                  # Declarative Jenkins build pipeline script
```

## 2. Local Development Setup

### Prerequisites
- Node.js (v22+)
- Docker and Docker Compose

### Run the Dev Stack
Spin up the PostgreSQL database and the NestJS application (with watch mode enabled and automatic database migrations):
```bash
docker-compose up --build
```
The NestJS application will be accessible at: `http://localhost:3000`.

### Database Migrations
Drizzle ORM handles database schemas programmatically.
- **Generate Migrations** (when schema changes in `src/database/schema.ts`):
  ```bash
  npm run db:generate
  ```
- **Apply Migrations**:
  ```bash
  npm run db:migrate
  ```

## 3. Automated Testing

### Unit Tests
Execute unit tests for services and controllers utilizing mocked Drizzle clients:
```bash
npm run test
```

### End-to-End (E2E) Integration Tests
The E2E tests spin up an isolated Postgres container (on port `5435`) and run assertions against real HTTP endpoints:
```bash
# 1. Spin up the test database
docker-compose -f docker-compose.test.yml up -d

# 2. Run migrations on the test database
DATABASE_URL=postgresql://postgres:postgrespassword@localhost:5435/tasks_test_db npm run db:migrate

# 3. Execute test cases
npm run test:e2e

# 4. Clean up test database
docker-compose -f docker-compose.test.yml down -v
```

## 4. Production Deployment

To run a production-optimized build using the multi-stage `Dockerfile`:
```bash
# Build the production Docker image
docker build -t nestjs-jenkins-app:latest .

# Run the production image (ensure DATABASE_URL environment variable is provided)
docker run -p 3000:3000 -e DATABASE_URL=postgresql://user:password@host:5432/db nestjs-jenkins-app:latest
```

## 5. Jenkins CI/CD Pipeline Setup

### Step 1: Install Jenkins
Run Jenkins locally inside a Docker container:
```bash
docker run -d \
  --name jenkins \
  -p 8080:8080 -p 50000:50000 \
  -v jenkins_home:/var/jenkins_home \
  -v /var/run/docker.sock:/var/run/docker.sock \
  jenkins/jenkins:lts
```

### Step 2: Install Jenkins Plugins
Log in to your Jenkins Dashboard (`http://localhost:8080`) and navigate to **Manage Jenkins** -> **Plugins** -> **Available Plugins** and install:
1. **Pipeline** (Core pipeline engine)
2. **Slack Notification** (For posting build results to Slack)
3. **Docker Pipeline** (For launching containers within pipeline steps)

### Step 3: Configure Slack Integration
1. Go to your Slack workspace and create a Slack App.
2. Under **Features**, enable **Incoming Webhooks** and click **Add New Webhook to Workspace**. Choose the channel (e.g. `#ci-cd-alerts`) and copy the Webhook URL.
3. In Jenkins, go to **Manage Jenkins** -> **Credentials** -> **System** -> **Global credentials**.
4. Click **Add Credentials**, choose **Secret text**, set the secret as your Slack webhook token, and give it the ID `SLACK_WEBHOOK_URL`.
5. Go to **Manage Jenkins** -> **System**. Scroll down to **Global Slack Notifier Settings**, configure the Slack Channel `#ci-cd-alerts` and set the Integration Token Credential ID.

### Step 4: Create a Pipeline Job
1. Click **New Item** on Jenkins homepage, enter the name `jenkins-test-app`, and select **Pipeline**.
2. Under the **Pipeline** configuration section, set **Definition** to `Pipeline script from SCM`.
3. Select **Git** as your SCM, and enter your Repository URL.
4. Set the **Script Path** to `Jenkinsfile` and save.
5. Click **Build Now** to execute the pipeline.
